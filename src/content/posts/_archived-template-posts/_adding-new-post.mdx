## 目录

## 创建博客文章

要撰写新的博客文章，请在 `src/content/posts/` 目录中创建一个 markdown （或 MDX）文件。

您可以将帖子组织到子目录中，以使内容更易于管理。子目录名称成为帖子 URL 的一部分。例如，`src/content/posts/2025/example-post.md` 将在 `/posts/2025/example-post` 可用。

如果您想要一个仅用于组织的子目录，而不影响 URL，请在文件夹名称前添加下划线 (`_`)。
```bash
# Example: post file paths and their URLs
src/content/posts/very-first-post.md          -> mysite.com/posts/very-first-post
src/content/posts/2025/example-post.md        -> mysite.com/posts/2025/example-post
src/content/posts/_2026/another-post.md       -> mysite.com/posts/another-post
src/content/posts/docs/_legacy/how-to.md      -> mysite.com/posts/docs/how-to
src/content/posts/Example Dir/Dummy Post.md   -> mysite.com/posts/example-dir/dummy-post
```
> 以 `_` 为前缀的文件和目录被排除在路由之外。将它们用于草稿、共享资源或仅供内部使用的内容。

## Frontmatter

Frontmatter 是存储博客文章元数据的主要位置。它以 YAML 格式位于文件的顶部。在 [Astro 文档](https://docs.astro.build/en/guides/markdown-content/) 中了解有关 frontmatter 及其用法的更多信息。

以下是每个帖子的 frontmatter 属性列表：

<ResponsiveTable variant="striped-minimal">

|属性 |描述 |备注 |
| ------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------- |
| **_标题_** |帖子的标题。 (h1) |必需<sup>\*</sup> |
| **_描述_** |帖子的描述。用于帖子摘录和帖子的网站描述。                                                     |必需<sup>\*</sup> |
| **_pubDatetime_** |以 ISO 8601 格式发布的datetime。                                                                                              |必需<sup>\*</sup> |
| **_modDatetime_** |采用 ISO 8601 格式的修改datetime。 （仅在修改博客文章时添加此属性） |可选|
| **_作者_** |帖子的作者。                                                                                                                 |默认 = `site.author` |
| **_精选_** |是否在主页的特色部分显示此帖子。                                                       |默认=假|
| **_草稿_** |将此帖子标记为"未发布"。                                                                                                    |默认=假|
| **_标签_** |这篇文章的相关关键词。以数组 YAML 格式编写。                                                                       |默认=其他|
| **_ogImage_** |帖子的 OG 图片。对于社交媒体共享和搜索引擎优化很有用。可以是远程 URL 或相对于当前文件夹的图像路径。 |默认 = `site.ogImage` 或生成的 OG 图像 |
| **_canonicalURL_** |规范 URL（绝对），以防文章已存在于其他来源上。                                                     |默认 = `Astro.site` + `Astro.url.pathname` |
| **_hideEditPost_** |隐藏帖子标题下方的编辑帖子按钮。仅适用于当前帖子。                                                   |默认=假|
| **_时区_** |为当前帖子指定 IANA 格式的时区。仅覆盖本文的全局 `site.timezone` 配置。             |默认 = `site.timezone` |

</ResponsiveTable>

> 提示！您可以通过在控制台中运行 `new Date().toISOString()` 来获取 ISO 8601 datetime。

仅需要指定 frontmatter 中的 `title`、`description` 和 `pubDatetime` 字段。

标题和描述（摘录）对于搜索引擎优化 (SEO) 非常重要，因此 AstroPaper 鼓励您在所有博客文章中包含这些内容。如果您在博客文章中省略 `tags`（换句话说，如果未指定标签），则默认标签 `others` 将用作该帖子的标签。您可以在`src/content.config.ts`中设置默认标签：
```ts file="src/content.config.ts"
// ...
tags: z.array(z.string()).default(["others"]), // replace "others" with whatever you want
// ...
```
### 示例 Frontmatter

这是帖子的示例 frontmatter。
```yaml file="src/content/posts/sample-post.md"
---
title: The title of the post
author: your name
pubDatetime: 2022-09-21T05:17:19Z
featured: true
draft: false
tags:
  - some
  - example
  - tags
ogImage: ../../assets/images/example.png # src/assets/images/example.png
# ogImage: "https://example.org/remote-image.png" # remote URL
description: This is the example description of the example post.
canonicalURL: https://example.org/my-article-was-already-posted-here
---
```
### VS 代码片段（可选）

AstroPaper 包含工作区片段以加快创建新帖子的速度：

- **frontmatter**：插入推荐的 frontmatter 块
- **模板**：插入基本帖子模板（包括`## Table of contents`）

这些片段位于 `.vscode/astro-paper.code-snippets` 中。如果您使用 VS Code（或 Cursor），它们应该在您打开工作区时自动可用。

## 添加目录

默认情况下，帖子不包含任何目录 (TOC)。要包含其中一个，请将 `Table of contents` 写为 h2 标题（Markdown 中的`##`）并将其放置在您希望其出现的位置：
```md
---
# frontmatter
---

Here are some recommendations, tips & tricks for creating new posts in AstroPaper blog theme.

<!-- [!code ++] -->
## Table of contents

<!-- the rest of the post -->
```
## 标题

关于标题，有一点需要注意。 AstroPaper 博客文章使用 `title` （来自 frontmatter）作为文章的主标题。因此，帖子中的其余标题应使用 `h2` \~ `h6`。

此规则不是强制性的，但出于视觉、可访问性和 SEO 目的强烈建议使用。

## 语法高亮

AstroPaper 使用 [Shiki](https://shiki.style/) 作为默认语法高亮器，使用 [@shikijs/transformers](https://shiki.style/packages/transformers) 来增强围栏代码块。如果您不想使用transformers，可以将其删除：
```bash
pnpm remove @shikijs/transformers
```

```ts file="astro.config.ts"
// ...
// [!code --:5]
import {
  transformerNotationDiff,
  transformerNotationHighlight,
  transformerNotationWordHighlight,
} from "@shikijs/transformers";

export default defineConfig({
  // ...
  markdown: {
    remarkPlugins: [remarkToc, [remarkCollapse, { test: "Table of contents" }]],
    shikiConfig: {
      themes: { light: "min-light", dark: "night-owl" },
      defaultColor: false,
      wrap: false,
      transformers: [
        transformerFileName(),
        // [!code --:3]
        transformerNotationHighlight(),
        transformerNotationWordHighlight(),
        transformerNotationDiff({ matchAlgorithm: "v3" }),
      ],
    },
  },
  // ...
});
```
## 存储博客内容的图像

以下是存储图像并在 Markdown 文件中使用它们的两种方法。

> 注意：如果您需要在 Markdown 中设置优化图像的样式，您应该[使用 MDX](https://docs.astro.build/en/guides/images/#images-in-mdx-files)。

### `src/assets/`目录内（推荐）

您可以将图像存储在 `src/assets/` 目录中。这些图像将由Astro通过[图像服务API](https://docs.astro.build/en/reference/image-service-reference/)自动优化。

您可以使用相对路径或别名路径（`@/assets/`）来引用这些图像。

示例：假设您要显示路径为 `src/assets/images/example.jpg` 的 `example.jpg`。
```md
![something](@/assets/images/example.jpg)

<!-- OR -->

![something](../../assets/images/example.jpg)

<!-- Using img tag or Image component won't work in markdown ❌ -->
<img src="@/assets/images/example.jpg" alt="something">
<!-- ^^ This is wrong -->
```
> 从技术上讲，您可以将图像存储在`src`下的任何目录中。 `src/assets`只是一个建议。

### `public/`目录内

您可以将图像存储在`public/`目录中。请记住，存储在 `public/` 中的图像不会受到 Astro 的影响，这意味着它们不会被优化，您需要自己处理图像优化。

对于这些图像，请使用绝对路径。它们可以使用 [markdown 图像语法](https://www.markdownguide.org/basic-syntax/#images-1) 或 HTML `img` 标签来显示。

示例：假设 `example.jpg` 位于 `public/assets/images/example.jpg`。
```md
![something](/assets/images/example.jpg)

<!-- OR -->

<img src="/assets/images/example.jpg" alt="something">
```
## 奖金

### 图像压缩

将图像放入博客文章时（尤其是`public/`目录中的图像），建议对其进行压缩。这将影响网站的整体性能。

推荐的图像压缩站点：

- [TinyPng](https://tinypng.com/)
- [TinyJPG](https://tinyjpg.com/)

### OG 图像

如果帖子未指定，则将使用默认的 OG 图像。尽管不是必需的，但应在 frontmatter 中指定与帖子相关的 OG 图片。 OG 图像的建议尺寸为 **_1200 X 640_** 像素。

> 从 AstroPaper v1.4.0 开始，如果未指定，OG 图像会自动生成。查看【公告】(https://astro-paper.pages.dev/posts/dynamic-og-image-generation-in-astropaper-blog-posts/)。
